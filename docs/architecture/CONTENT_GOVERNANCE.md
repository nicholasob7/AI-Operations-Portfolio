# Content Governance

## Purpose

This document defines the accepted doctrine for how public truth is modeled, classified, repeated, and projected across the site.

Its purpose is to prevent drift, duplication, and accidental elevation of projection prose into separate truth.

## Hybrid Layered Truth Model

The accepted truth model is hybrid layered.

Canonical truth is composed of:

- entities
- facts
- relationships
- evidence/support records
- governed prose blocks only where wording carries irreducible public meaning

Projection layers are separate from canonical truth.

UI copy sits outside truth acquisition unless it contains direct public facts.

## Canonical Backbone

### Entities

The public things the site is about.

Examples include:

- person
- project
- skill domain
- qualification
- contact channel
- public artifact

### Facts

Normalized public claims about entities or relationships.

Facts answer:

- what is true?

### Relationships

Links between entities and facts.

Relationships define how the public truth connects across the site.

### Evidence / support records

Substantiating records tied to facts.

Evidence answers:

- what supports or demonstrates that truth?

Evidence is not the same thing as the fact it supports.

### Governed prose blocks

Governed prose is canonical only by exception.

A prose block qualifies only when its wording itself carries irreducible public meaning that would be materially damaged by reduction into normalized facts, relationships, and evidence.

Projection is the default.
Governed prose is the exception.

## Projection Layers

Projection layers express canonical truth for particular reading conditions.

Accepted projection roles include:

- summary
- detail
- support
- navigation/reference

Projection prose is not a separate truth object merely because it appears on a page.
Projection prose may remain publicly important and preserved, but it does not become canonical truth merely by being public.

## UI / Presentation Copy

UI and presentation copy is outside truth acquisition unless it directly expresses a public fact.

Examples include:

- button labels
- toggle text
- section controls
- action labels
- view-state wording

UI copy must not be treated as canonical truth merely because it is visible.

## Deduplication Doctrine

### Same fact once, many projections

If the same public fact appears across multiple pages or surfaces, it remains one canonical fact.

Repeated appearance does not create new truth.

### Summary is not a new truth object

A summary of deeper content is a projection, not a separate canonical record.

### Evidence is not fact

A supporting item may strengthen, qualify, or demonstrate a fact, but it does not replace the fact and should not be modeled as duplicate truth.

### Provenance and wording variants

Wording variants and page-occurrence provenance may be preserved as secondary/contextual material.

They are not primary truth unless they meet the governed-prose threshold.

## Anti-Drift Rule

Web, agent, and PDF / print outputs must not be separately authored truths.

Different surfaces may differ in:

- density
- sequencing
- visual pacing
- interaction
- layout

They must not differ by maintaining separate underlying public truth.

## Public Retrieval Rule

Every intentionally public word must be recoverable by agents from a coherent labeled structure derived from canonical truth.

This does not require flattening the human UX.

It does require that public truth remain machine-recoverable without depending on imitation of human interaction as the only acquisition path.

## Governance Priority

When there is tension between:

- convenience of page-local prose authoring
- and preservation of coherent public truth

coherent public truth takes priority.
